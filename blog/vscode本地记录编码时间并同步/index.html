vscode本地记录编码时间并同步

Authors
  • Name
    Later
    Twitter

背景

我想记录下我的编码时间，以及各种数据统计，在vscode的拓展市场里找了下，两个有名的都不符合我的要求：

  1. WakaTime

虽然有每周和每日的电子邮件报告，但是免费版本的只有两周的数据记录

  1. Code Time 免费版本的只保留90天的数据记录

我的要求很简单，就是我一直可以看到我的编码统计数据，数据还要同步，虽然我大部分时间在公司编码，但是我有时候也在家里编码。

Coding Tracker + Gist + 手动

找了一圈找到一个Coding Tracker插件，在拓展商店的点赞还不少(7k)，看来是稳定可靠的，但是这个插件只是一个客户端，所有生成的数据都保存在本地的~/.coding-tracker目录下，可以选择服务端模式，但是需要假设配套的服务器，比较麻烦。

Gist 支持操作gitee(码云)和github gist操作的vscode插件

思路：

  1. 先用Coding Tracker将编码时间记录到本地
  2. 在gitee上创建私有仓库专门保存Coding Tracker在~/.coding-tracker目录下生成的数据
  3. 写两个脚本，每次换电脑时上传或者下载下自己的编码数据覆盖到~/.coding-tracker
  4. 上传和下载的代码片段可以保存在gitee的gist里，可以用Gist插件 上传：
cd ~/.coding-tracker


rm -rf .git/


git init


git add .


git commit -m "update"


git remote add origin https://gitee.com/hou1123/coding-tracker-data.git


git push origin master -f

下载：

cd ~/


mv .coding-tracker/ .coding-tracker-back/


git clone https://gitee.com/hou1123/coding-tracker-data.git .coding-tracker
