git rebase 合并多个 commit

Authors
  • Name
    Later
    Twitter
  1. git rebase -i [startpoint] [endpoint]

-i 的含义是：--interactive, 即弹出交互式的界面让用户编辑完成合并操作 [startpoint] 指的是合并区间的起点。 [endpoint] 指的是合并区间的终点，默认是当前分支 HEAD 所指向的 commit。 这里的区间是一个前开后闭的区间。

  1. 进入编辑框完成操作

p, pick: 保留该 commit。 r, reword: 保留该 commit，可以修改 commit 的注释。 e, eidt: 保留该 commit，但停下来修改该 commit （不仅仅是注释），可以用来解决 merge 冲突。 s, squash: 将该 commit 和 前面一个 commit 合并。 f, fixup: 将该 commit 和 前面一个 commit 合并，但不保留该提交的注释信息。 x, exec: 执行 shell 命令。 d, drop: 丢弃该 commit。

编辑 commit 内容。将 “pick” 改成 “squash” 或者 “s”，意思是将该 commit （add b.php）和 前面的 commit (add a.php) 合并。

  1. 修改合并之后的 commit 注释

4. 此时进入了一个临时的分支，基于临时分支创建一个新分支

git checkout -b temp

5. 切回到原分支执行rebase

git checkout current_branch && git rebase temp

  1. 完成，强制提交

2021-08-07: 最新版的git不用切分支了，直接提交即可
