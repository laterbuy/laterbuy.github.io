node实现一个简易git-init

Authors
  • Name
    Later
    Twitter

背景

前几周一直阅读gitlet的源码，读到最后一头雾水，所以又去读了progit内部原理部分，才明白了一些git内部原理。

之前想着读完gitlet的源码理解下git内部原理就可以了，但是发现gitlet的实现不是从易于理解源码的角度出发的，而且它内部的一些实现并没有帮助易于理解源码，反而使人更加迷糊，比如它把commitId和commit信息又做了一次hash,每次checkout用改hash做处理，完全让人更加摸不着头脑。

由于我已经基本阅读完了gitlet的源码，所以我决定写一系列文章，实现一个更简易的git，完全从易于理解原理的角度出发，剔除调一切不必要的功能。

源码简单实现

我们随便新建一新个目录，执行git init命令，然后进去cd .git目录，列出当前目录的完整内容tree

我们只关注四个重要的文件或目录 HEAD 及 index 文件，objects 及 refs 目录。这些是 Git 的核心部分。objects 目录存储所有数据内容，refs 目录存储指向数据 (分支) 的提交对象的指针，HEAD 文件指向当前分支，index 文件保存了暂存区域信息

├── HEAD
├── objects
└── refs
    └── heads

所以执行git init时， 我们可以初始化这四个重要的文件或目录即可，我们将这四个目录放在.ngit下， 这里边只有HEAD是文件（内容为ref: refs/heads/master），其他两个都是目录，index文件会在执行git add path时生成，可以暂时先不考虑。

const fs = require('fs');
const path = require('path');
/**
 * @description:  根据对象写文件或者目录
 * @param {*}
 * @return {*}
 */
const writeFile = (initGitFile, dir) => {
  Object.keys(initGitFile).forEach((key) => {
    if (typeof initGitFile[key] === 'string') {
      fs.writeFileSync(path.resolve(dir, key), initGitFile[key]);
    }
    if (typeof initGitFile[key] === 'object') {
      const dirName = path.resolve(dir, key);
      fs.mkdirSync(dirName);
      writeFile(initGitFile[key], dirName);
    }
  });
};


const init = () => {
  // 声明.ngit目录
  const gitDir = path.resolve(process.cwd(), '.ngit');
  // 判断当前路径是否已经在.ngit目录
  const isExistingGit = fs.existsSync(gitDir);
  // 已经在.ngit目录直接返回
  if (isExistingGit) {
    return `Reinitialized existing Git repository in ${gitDir}`;
  }
  // 初始文件目录结构
  const initGitFile = {
    HEAD: 'ref: refs/heads/master\n',
    objects: {},
    refs: {
      heads: {},
    },
  };
  // 先.ngit写目录
  fs.mkdirSync(gitDir);


  // 直接写文件和目录
  writeFile(initGitFile, gitDir);


  return `Initialized empty Git repository in ${gitDir}`;
};
